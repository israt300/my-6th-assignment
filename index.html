Frequently Asked Questions

1. The difference between var, let, and const ?

var: Function-scoped and can be redeclared.let: Block-scoped and cannot be redeclared.const: Block-scoped and its value cannot be changed once assigned.

2. The difference between map(), forEach(), and filter() ?

map(): Applies a function on each item of the array and creates a new array.forEach(): Runs a function on each item but doesn't return anything.filter(): Creates a new array with only the items that meet a condition.

3. Explain arrow functions and how they are different from regular functions ?

Arrow functions are a shorter syntax for writing functions in JavaScript, using the => operator. They don't have their own this, so they inherit it from the surrounding context. In contrast, regular functions have their own this, which can lead to different behavior when used inside objects or callbacks.

4. How JavaScript Promises work?

JavaScript Promise is an operation that will complete sometime in the future. When the operation is successful, we use .then(), and if there is an error or failure, we handle it with .catch(). It helps in managing asynchronous tasks more easily.

5. How closures work in JavaScript?

Closures in JavaScript occur when a function retains access to its lexical scope, even after the outer function has finished executing. This means that an inner function can remember and access variables from its outer function, even after the outer function has returned.
